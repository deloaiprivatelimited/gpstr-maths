# INPUT CONTEXT

ಮೂಲ ವಿಷಯ ಕೆಳಗಿನ ರೂಪದಲ್ಲಿರಬಹುದು:

• ಒಂದು ಅಥವಾ ಹೆಚ್ಚಿನ PDF ಭಾಗಗಳು
• ವಿಭಿನ್ನ ತರಗತಿಯ ಗಣಿತ ಪಠ್ಯಪುಸ್ತಕಗಳಿಂದ ತೆಗೆದ ಭಾಗಗಳು
• ಕೆಲವು ಸಂದರ್ಭಗಳಲ್ಲಿ ಒಂದೇ ಅಧ್ಯಾಯವನ್ನು ಹಲವು ಮೂಲಗಳಿಂದ ಸಂಯೋಜಿಸಲಾಗಿದೆ

PDF ಭಾಗಗಳು **page-range extracts** ಆಗಿರಬಹುದು.

ಆದ್ದರಿಂದ:

• PDF ನಲ್ಲಿ ಕಾಣದ ವಿಷಯಗಳನ್ನು Missing ಎಂದು ಗುರುತಿಸಬಾರದು
• ನೀಡಲಾದ ಮೂಲ ಭಾಗಗಳ ಆಧಾರದ ಮೇಲೆ ಮಾತ್ರ ಮೌಲ್ಯಮಾಪನ ಮಾಡಬೇಕು

# MERGED SOURCE VALIDATION RULE

ಒಂದಕ್ಕಿಂತ ಹೆಚ್ಚು ಪಠ್ಯಪುಸ್ತಕ ಮೂಲಗಳನ್ನು ಸಂಯೋಜಿಸಿದ ಅಧ್ಯಾಯವಾಗಿದ್ದರೆ:

• ಒಂದೇ theorem ಅಥವಾ formula ಮರುಕಳಿಸದಿರುವುದು ಸರಿಯಾದ ಕ್ರಮ
• ಒಂದೇ worked example ಮರುಬಳಕೆ ಆಗದಿರುವುದು ಸರಿಯಾದ ಕ್ರಮ
• synthesis explanations ಅನ್ನು Missing ಎಂದು ಗುರುತಿಸಬಾರದು
• duplicate removal ಕಾರಣಕ್ಕೆ Missing flag ಕೊಡಬಾರದು

# VALIDATION CRITERIA

## 1️⃣ Content Coverage Audit

ಪರಿಶೀಲಿಸಬೇಕು:

* ಎಲ್ಲಾ ಪ್ರಮುಖ **Definitions** ಸೇರಿಸಲಾಗಿದೆವೆ?
* ಎಲ್ಲಾ ಪ್ರಮುಖ **Formulas** ಸೇರಿಸಲಾಗಿದೆವೆ?
* ಎಲ್ಲಾ ಪ್ರಮುಖ **Theorems / Properties** ಸೇರಿಸಲಾಗಿದೆವೆ?
* ಎಲ್ಲಾ ಮುಖ್ಯ **sub-concepts / headings** cover ಆಗಿವೆವೆ?
* Worked examples logical intent ಉಳಿದಿದೆಯೆ?

Missing sections ಸ್ಪಷ್ಟವಾಗಿ ಗುರುತಿಸಬೇಕು.

---

# 2️⃣ Mathematical Accuracy Audit

ಈ ಅಂಶಗಳನ್ನು ಪರಿಶೀಲಿಸಬೇಕು:

* Formula syntax ಸರಿಯಾಗಿದೆಯೆ?
* LaTeX representation ಸರಿಯಾಗಿದೆಯೆ?
* theorem statement ಸರಿಯಾಗಿದೆಯೆ?
* proof sketch ತಪ್ಪಾಗಿದೆಯೆ?
* example solution steps logically valid ಆಗಿವೆಯೆ?
* boundary conditions ತಪ್ಪಾಗಿದೆಯೆ?

ಕೆಳಗಿನ ದೋಷಗಳು **CRITICAL**:

• ತಪ್ಪಾದ formula
• theorem statement distortion
• wrong derivation
• incorrect example result
• incorrect algebra manipulation

---

# 3️⃣ Concept Integrity Audit

ಪರಿಶೀಲಿಸಬೇಕು:

* ಮೂಲ ಗಣಿತ ಕಲ್ಪನೆಯ ಅರ್ಥ ಬದಲಾಗಿದೆಯೆ?
* theorem / property conceptual distortion ಇದೆಯೆ?
* important condition ಅಥವಾ restriction ಕಳೆದುಹೋಗಿದೆಯೆ?

ಉದಾಹರಣೆ:

Pythagoras theorem condition missing → distortion

---

# 4️⃣ Worked Example Validation

ಪರಿಶೀಲಿಸಬೇಕು:

* steps logically follow ಆಗುತ್ತಿವೆಯೆ?
* justification valid ಆಗಿದೆಯೆ?
* final answer correct ಆಗಿದೆಯೆ?
* example solution original concept demonstrate ಮಾಡುತ್ತಿದೆಯೆ?

ಕೆಳಗಿನವು **CRITICAL**:

• wrong result
• missing key step
• incorrect algebra manipulation
• theorem misuse

---

# 5️⃣ Diagram & Table Validation

Math prompt ನಲ್ಲಿ diagram inclusion rules ಇದ್ದು:

Diagram/Table include ಆಗಬೇಕಾದರೆ:

CONDITION 1
Text ಅಥವಾ formula ಮೂಲಕ ಮಾತ್ರ explain ಮಾಡಲಾಗದು

CONDITION 2
Exam context ನಲ್ಲಿ ಬರುತ್ತದೆ

Validation ಸಮಯದಲ್ಲಿ ಪರಿಶೀಲಿಸಬೇಕು:

* unnecessary diagram ಸೇರಿಸಲಾಗಿದೆವೆ?
* required diagram miss ಆಗಿದೆಯೆ?
* diagram JSON structure valid ಆಗಿದೆಯೆ?
* exam_relevance field ಇದೆವೆ?
* cannot_be_expressed_in_text flag correct ಆಗಿದೆಯೆ?

Diagram ID rule:

class{X}*math_ch{Y}*{concept}_{graphtype}

---

# 6️⃣ Structural Compliance Audit

ಪರಿಶೀಲಿಸಬೇಕು:

* [ಅಧ್ಯಾಯ_ಮೆಟಾಡೇಟಾ] block ಇದೆವೆ?
* concept_id format ಸರಿಯಾಗಿದೆಯೆ?
* required blocks ಎಲ್ಲಿವೆವೆ?

Required blocks:

[ವ್ಯಾಖ್ಯೆ]
[ಸೂತ್ರಗಳು]
[ಪ್ರಮೇಯ]
[ಅಂಶಪಟ್ಟಿ / Properties]
[ಕ್ರಮಬದ್ಧ ಉದಾಹರಣೆ]
[ವಿಶ್ಲೇಷಣೆ]
[ಸಂಬಂಧ]
[ಶೈಕ್ಷಣಿಕ_ದೃಷ್ಟಿಕೋನ]

Content_Coverage_Check block validate ಮಾಡಬೇಕು.

# ISSUE CLASSIFICATION FRAMEWORK

## CRITICAL

* ತಪ್ಪಾದ formula
* theorem distortion
* wrong example result
* algebra / derivation error
* missing major concept
* structural collapse
* invalid LaTeX formula

---

## MAJOR

* theorem incomplete
* example step incomplete
* diagram JSON incorrect
* table structure incorrect
* concept explanation partially missing

---

## MINOR

* notation inconsistency
* unclear wording
* formatting issue

---

## SAFE

Release ಗೆ ತಯಾರಾದ ಸ್ಥಿತಿ
